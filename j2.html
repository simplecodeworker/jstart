Identifiers, Variables & Constants

Identifiers

Technical term for a name in a programming language used to name classes, methods, variables, etc.

  • Cannot start with a digit.
  • Cannot contain spaces.
  • Cannot be a reserved keyword (like class, int).
Valid
  • itemCost
  • _itemCost
  • item$Cost
  • itemCost2
Invalid
  • item#Cost (no #)
  • item-Cost (no -)
  • 2itemCost (starts w/ digit)
  • abstract (keyword)

Variables

The name of a memory location used to store data. Its value may change during execution.

Constants

Declared with the keyword final. Its value cannot be changed once assigned.

Literals

Literals are fixed values directly written into the code. They are the constant values assigned to variables.

int x = y + 30_000; // int literal (underscores allowed for readability)
System.out.println('A'); // char literal (MUST use single quotes)
boolean finished = false; // boolean literal
long g = 9; // long literal (implicit cast from int literal 9)
int binary = 0b1001; // binary literal (prefix with 0b)

Assignment Statements

The Biggest Beginner Mistake

In Java, = does NOT mean mathematical equality.
It is the assignment operator. It means "calculate the expression on the right, and store it in the variable on the left."

variable = expression;

Expressions

An expression can be a value, a variable, or a computation.

Type Matching constraint

The data type of the variable must match (or be able to safely hold) the expression's type.
