What is Debugging?

In programming, a bug is an error or flaw that causes a program to produce an incorrect or unexpected result.

"Debugging is like being the detective in a crime movie where you are also the murderer."

Debugging is the process of finding and fixing these errors. Instead of guessing why code doesn't work, we use tools to watch the code run one line at a time.

The Breakpoint

What is a Breakpoint?

A breakpoint is a "stop sign" you place on a specific line of code. When you run your program in Debug Mode, Eclipse will pause execution right before that line runs.

How to set one:

  • • Double-click the blue margin (gutter) on the left of the line number.
  • • A ● icon appears.

Stepping Through Code

Once paused, you have three primary "moves" to control how the program proceeds.

Step Over (F6)

Executes the current line and moves to the next line in the same method. Use this 90% of the time.

Step Into (F5)

If the current line calls a method, "Step Into" takes you inside that method's code.

Step Return (F7)

Finishes the current method and takes you back to the line that called it (the caller).

Cleaning Up

Terminating the Debugger

Don't leave debug sessions hanging! If you found the bug or want to stop, use the Terminate button.

The Red Square

Resume vs Terminate

Resume (F8)

Keep running until the next breakpoint is hit.

Terminate

Kill the program execution entirely.
