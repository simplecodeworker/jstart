Q6.3.1 & Q6.3.2 - Object Oriented Programming

Q6.3.1: Donation Class Design

Design a Donation class with public instance variables (donor, amount), a static property (totalAmount), a constructor, and a donate() method. 
public class Donation {
    public String donor;
    public double amount;
    public static double totalAmount; // Shared across all instances

    public Donation(String name, double initialAmt) {
        this.donor = name;
        this.amount = initialAmt;
        totalAmount += initialAmt;
    }

    public void donate(double amt) {
        this.amount += amt;
        totalAmount += amt;
    }
}
Beginner Tip: Note the use of the static keyword. It makes totalAmount belong to the class rather than a specific object, meaning every donation updates this single central value.

Q6.3.2: Testing Program

Create a test program to handle donations for "Mr. Lee KC" and "Foundation Two", then display individual and total sums. 
public class Main {
    public static void main(String[] args) {
        Donation lee = new Donation("Mr. Lee KC", 100000);
        lee.donate(8000);
        lee.donate(5000);

        Donation org = new Donation("Foundation Two", 10000);
        org.donate(5000);
        org.donate(3000);
        org.donate(9000);

        System.out.println("Lee Total: " + lee.amount);
        System.out.println("Org Total: " + org.amount);
        System.out.println("Grand Total: " + Donation.totalAmount);
    }
}
Beginner Tip: Access static members like totalAmount using the Class name (Donation.totalAmount), not the object variable. This is standard Java practice to show that the variable is static.
