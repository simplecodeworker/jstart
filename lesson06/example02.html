Understanding Java Accessors

A visual guide to Encapsulation: Protecting data with private variables and public methods.

Part 1

The Blueprint: CircleWithAccessors.java


// 1. Package declaration
package oop6TestCircleWithAccessors;

// 2. The Class definition
public class CircleWithAccessors {
  
  // 3. Private data (Encapsulation)
  private double radius;

  // 4. No-arg constructor
  public CircleWithAccessors() {
    radius = 1.0;
  }

  // 5. Constructor with parameter
  public CircleWithAccessors(double r) {
    radius = r;
  }

  // 6. The Getter (Accessor)
  public double getRadius() {
    return radius;
  }

  // 7. The Setter (Mutator)
  public void setRadius(double newRadius) { 
    radius = newRadius;
  }

  // 8. Method for logic
  public double findArea() {
    return radius * radius * 3.14159;
  }
}

1 & 2. The Setup

We define a container for our circle logic. Everything inside the curly braces belongs to this "Blueprint".

3. Private Variable (The Core Concept)

The private keyword is a lock. It prevents other classes from touching the radius directly. This is called Encapsulation.

4 & 5. Constructors

These are "birth certificates" for objects. One creates a default circle (radius 1.0), and the other lets you choose the size when you create it.

6 & 7. Accessors (Getters & Setters)

Because radius is private, we provide these public "windows". getRadius lets people see the value, and setRadius lets them change it safely.

8. Functional Method

A public action that uses the internal data to perform a calculation (calculating Area).

Part 2

The Execution: TestCircleWithAccessors.java


public class TestCircleWithAccessors {
  public static void main(String[] args) {

    // A. Create object with radius 5.0
    CircleWithAccessors myCircle = 
        new CircleWithAccessors(5.0);

    // B. Using the Getter to read data
    System.out.println("Radius is " + 
        myCircle.getRadius());

    // C. Trying direct access (ERROR!)
    // myCircle.radius = 10; // This won't compile!

    // D. Using the Setter to update data
    double currentRadius = myCircle.getRadius();
    myCircle.setRadius(currentRadius * 1.1);

    // E. Final calculation
    System.out.println("New Area: " + 
        myCircle.findArea());
  }
}

A. Object Instantiation

We create an instance of our blueprint. We call the constructor and pass 5.0 to set the initial radius.

B. Reading Private Data

We cannot type myCircle.radius. We must ask politely using getRadius(). This ensures the class stays in control of its data.

C. Why the Error?

If you uncommented that line, Java would stop you! "radius has private access." This prevents accidental data corruption from outside classes.

D. Updating via Mutator

To change the radius (like increasing it by 10%), we use the setRadius() method. Inside the class, we could even add logic to prevent negative numbers here!

Key Takeaway for Beginners

Private Variables

Keep data safe and hidden.

Getters (Accessors)

Provide read-only access to data.

Setters (Mutators)

Allow controlled modification of data.
