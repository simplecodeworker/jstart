Section 2

Understanding Access Control

Access modifiers control the visibility of variables and methods. Think of them as security clearance levels.

Modifier Accessible in Class? Accessible by World?
public Yes (Y) Yes (Y)
private Yes (Y) No (N)

Public Modifier

The member is accessible to ALL other objects in the system. Use it for the interface you want others to use.

Private Modifier

The member is accessible within this class only. This is the cornerstone of data security.

Takeaway

Use 'private' to hide data and 'public' to provide a controlled interface.

Section 3

Enforcing Data Privacy

What happens when you break the rules? Java enforces privacy at Compile-Time. The compiler stops you from even starting the program if you attempt to touch private data from outside.

Circle.java & testCircle.java Compilation Error 
class Circle {
  private double radius = 1.0;
}

public class testCircle {
  public static void main(String[] args) {
    Circle c = new Circle();

    System.out.println(c.radius); // ❌ radius has private access in Circle
  }
}
Why the failure?

The radius is shielded. Even though c is a Circle object, testCircle is a different class and does not have permission to look inside.

Section 4

Accessor Methods

If data is private, how do we use it? We use Accessor Methods (Getters and Setters). They act as a controlled "middleman" between the outside world and the private data.

Getter (Accessor)

Retrieves the value of a private variable without allowing direct modification.

public double getRadius() {
  return radius;
}
Setter (Mutator)

Changes the value of a private variable, often adding validation logic.

public void setRadius(double r) {
  this.radius = r;
}

Section 6

Constructor Requirements

Constructors define the "required ingredients" to create an object. If you provide a custom constructor, Java assumes you want total control.

The Default Constructor Warning

"If you define a constructor that requires a parameter, you can no longer use the default 'no-arg' constructor unless you explicitly write one."

public class Test {
  public static void main(String[] args) {
    Circle c1 = new Circle(3.5); // ✅ Works
    Circle c2 = new Circle();    // ❌ ERROR: no constructor match
  }
}

Section 7

Multiple Constructors

For flexibility, a class can have multiple constructors. This allows a programmer to create a "default" object or a "customized" one.

Path A: Default 
public Circle() {
  this.radius = 1.0;
}
Path B: Custom 
public Circle(double r) {
  this.radius = r;
}

Section 8

Method Overloading

Having more than one constructor with the same name is called **Overloading**. The compiler identifies which one to call based on the **Signature**.

What is a Signature?

It is the combination of:
1. Number of params
2. Types of params
3. Order of types.

Variable names do NOT count!

Section 13

Garbage Collection

Java manages memory for you. The **Garbage Collector** (GC) automatically deletes unreachable objects to reclaim space.

The Definition of Garbage

Any object in memory that no longer has any reference pointing to it is "Garbage".

Correct Object Cloning

To copy data correctly, copy fields individually:
c1.setRadius(c2.getRadius())
