Java: Static vs. Instance Variables

Understanding how objects share data using the static keyword.

The Blueprint: CircleWithStaticVariable.java

Java Class Code 
public class CircleWithStaticVariable {
  /** The radius (Instance Variable) */
  private double radius;

  /** The count (Static Variable) */
  private static int numOfObjects = 0;  
  
  public final double PAI = 3.14;

  public CircleWithStaticVariable() {
    radius = 1.0;
    numOfObjects++;
  }

  public CircleWithStaticVariable(double r) {
    radius = r;
    numOfObjects++;
  }

  public static int getNumOfObjects() {
    return numOfObjects;
  }
  
  // ... methods for getRadius, setRadius, findArea
}

The Instance Variable (radius)

Every time you use new, a unique copy of radius is created in memory. If you have 100 circles, you have 100 different radius values stored in 100 different locations.

The Static Variable (numOfObjects)

The keyword static means this variable belongs to the Class itself, not any specific object. There is only one copy of this variable shared by every circle object ever created.

The Execution: TestCircleWithStaticVariable.java

Application Code 
public class TestCircleWithStaticVariable {
  public static void main(String[] args) {
    // 1. Create circle1
    CircleWithStaticVariable circle1 = new CircleWithStaticVariable();
    
    System.out.println("Circle 1 count: " + circle1.getNumOfObjects());

    // 2. Create circle2
    CircleWithStaticVariable circle2 = new CircleWithStaticVariable(5);

    // 3. The "Why?" - Observe the difference
    System.out.println(circle2.getNumOfObjects()); // Result: 2
    System.out.println(circle1.getNumOfObjects()); // Result: 2
    System.out.println(CircleWithStaticVariable.getNumOfObjects()); // Result: 2
    
    // 4. Modify instance variable
    circle1.setRadius(9);
    
    // Display results
    System.out.println("Circle 1 radius: " + circle1.getRadius());
    System.out.println("Circle 2 radius: " + circle2.getRadius());
  }
}
Step 1 & 2: Object Creation

When circle1 was created, numOfObjects became 1.
When circle2 was created, the same variable was incremented to 2.

Step 3: The "Why?" Explained

Why do all three print statements output "2"?

  • 1. circle1 and circle2 are just pointing to the same shared count in the class memory.
  • 2. Even though we call it via circle1, Java internally treats it as CircleWithStaticVariable.getNumOfObjects().
Step 4: Instance Autonomy

When we set circle1 radius to 9, the radius of circle2 remained 5. This is because Instance Variables are private and unique to each object.

Quick Cheat Sheet

Instance Variables

  • • One copy per object
  • • Used for unique properties (like a circle's radius)
  • • Accessed via: objectName.variable

Static Variables

  • • One copy per class (shared)
  • • Used for common data (like a global counter)
  • • Accessed via: ClassName.variable
