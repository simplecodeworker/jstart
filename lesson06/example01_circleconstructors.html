Circle Class with Constructors

Part 1

The Blueprint: The Class

This code defines the "Idea" of a circle. It's not a specific circle yet—it's the set of rules for making one.

package oop6TestCircleWithConstructors;

// Define the Circle class with two constructors
class CircleWithConstructors {
   double radius;

   /** no-arg constructor */
   CircleWithConstructors() {
     radius = 1.0;
   }

   /** Construct a circle with a specified radius */
   CircleWithConstructors(double r) {
      radius = r;
   }

   /** Return the area of this circle */
   double findArea() {
     return radius * radius * 3.14159;
   }
}

The Data Field (State)

The double radius variable is where every Circle object will store its size. In OOP, we call this the "State" of the object.

No-Arg Constructor

Think of this as the "Default Builder." If you don't provide a size, Java uses this to create a standard circle with a radius of 1.0.

Parameterized Constructor

The "Custom Builder." It allows you to specify a value (r) at the exact moment the circle is born, setting the radius immediately.

The Method (Behavior)

findArea() is what the circle can *do*. It uses the circle's own radius to calculate and return the area.

Part 2

The Reality: The Application

This is the workshop where we actually use the blueprint to create living objects in memory.

package oop6TestCircleWithConstructors;
import java.util.Scanner;

public class TestCircleWithConstructors {
  public static void main(String[] args) {
    Scanner s = new Scanner(System.in); 

    // 1. Create a Circle with radius 8.0
    CircleWithConstructors myCircle = 
        new CircleWithConstructors(8.0);
    
    // 2. Create a Circle with default radius
    CircleWithConstructors yourCircle = 
        new CircleWithConstructors();

    // 3. Modify circle radius
    yourCircle.radius = 100;

    System.out.println("Area: " + yourCircle.findArea());
  }
}

Instantiating Objects

Using the keyword new is like saying "Execute the builder!" This allocates space in your computer's memory for a real object.

Two Distinct Lives

myCircle and yourCircle are completely separate. Changing the radius of one does not change the other.

The Dot Operator

The . is how we reach inside an object. yourCircle.radius allows us to access or change the specific data stored inside that object.

🍕

The Object

The "Instance" or "The Meal". It is the actual entity created in memory using the recipe.
