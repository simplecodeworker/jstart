1. What is UML?

Unified Modelling Language (UML) is a general-purpose modelling language used in software engineering.

  • Not a programming language: It's a visual language.
  • The Goal: To define a standard way to visualize the structure and behavior of a system.
  • For Everyone: Helps engineers, architects, and business stakeholders communicate effectively.

"Think of UML as the architectural blueprint for a house before the builders start laying the bricks (writing the code)."

Takeaway: Types

UML often shows types like balance: double. In Java, the type comes first: double balance;.

Takeaway: Visibility Modifiers

In advanced UML, you will see symbols like + or - before names to dictate who can access them. We will dive into this next!

4. Visibility Modifiers: Who Can See What?

In Object-Oriented Programming, we use access modifiers to protect our data (a concept called Encapsulation). UML uses simple symbols in front of attributes and methods to represent these modifiers.

+

Public

Accessible from anywhere. Visible to everyone.

-

Private

Accessible ONLY inside the class itself. Highly restricted.

#

Protected

Accessible to subclasses and classes in the same package.

~

Package (Default)

Accessible only to classes in the same package.

BankAccount
-balance: double
-accountNumber: String
#branchCode: String
~status: int
+ownerName: String

5. Relationships: Connecting Classes

Software isn't just a list of separate classes; they need to talk to each other. We use Associations to show these links.

Example: Student & School

School
+ name: String
+ enroll()
"belongs to"
Student
- id: int
+ name: String
+ study()

In UML

A simple line connects the two classes. It often has a label (like "belongs to") or an open arrow to show which class "knows" about the other.

In Java

The relationship is represented as a Reference Variable inside one of the classes.

public class Student {
  School mySchool; // The Association!
}
Pro Tip: "Has-A" Relationship

Associations are often called "Has-A" relationships. A Student has a School. An Employee has a Manager.
