Section 2: Identifying Code Redundancy in Object Design

Imagine a scenario for a School Library serving two types of members: Students and Staff. Let's look at how we might initially design their classes:

Class: Student

Attributes:

  • name, ID, department, quota
  • studentClass

Methods:

  • setBasics(), setQuota(), getQuota()

Class: Staff

Attributes:

  • name, ID, department, quota
  • designation

Methods:

  • setBasics(), setQuota(), getQuota()
Key Takeaway: Notice the redundancy! When different entities share many common characteristics (like name, ID, and department), duplicating class definitions leads to redundant, hard-to-maintain code.

Section 3: Designing a Superclass for Code Reuse

We can propose an improved design using inheritance. A parent class "Member" holds common characteristics, while "Student" and "Staff" become subclasses.

Member (Superclass)
Attributes: name, ID, department, quota
Methods: setBasics(), setQuota(), getQuota()
Student (Subclass)
Added: studentClass
Staff (Subclass)
Added: designation
Key Takeaway: Inheritance allows for code reuse and better organization. By defining common attributes and methods in a "superclass," subclasses can inherit them automatically. Arrows in UML point from Subclass to Superclass.

Section 4: Establishing the Is-A Relationship

Once defined as subclasses, Student and Staff automatically inherit the characteristics of Member without re-specifying them. They can also define their own unique characteristics.

🧬

Inherited Traits:

  • Attributes: name, ID, department, quota
  • Methods: setBasics(), setQuota(), getQuota()
Key Takeaway: Inheritance enables "is-a" relationships (e.g., A Student "is-a" Member), where a subclass is a specialized version of the superclass. This promotes cleaner code maintenance.

Section 5: Implementing Inheritance in Java with the Extends Keyword

Let's look at the actual code implementation of the concept using Java.

class Member {
    String name;
    String ID;
    String department;
    private int quota;

    void setBasic(String n, String i, String d) {
        name = n;
        ID = i;
        department = d;
    }
    void setQuota(int q) { quota = q; }
    int getQuota() { return quota; }
}

class Student extends Member {
    String studentClass;
}

class Staff extends Member {
    String designation;
}
Key Takeaway: The keyword extends is the core mechanism in Java to create a subclass. The subclass automatically gains access to the non-private members of the superclass.

Section 7: Inheritance in Constructors and Field Access

Let's look at another scenario with Father and Son classes to understand constructors and attribute access.

class Father {
    private double salary; // Private!
    String surName = "Chee";
    String givenName;

    public Father() { // Constructor
        givenName = "Senior George";
        salary = 2000;
    }
    
    public double getSalary() {
        return salary;
    }
}

class Son extends Father {
    private int score;

    public Son() { // Constructor
        score = 80;
        givenName = "Little George"; // Accessing parent's non-private variable
    }
    
    public int getScore() {
        return score;
    }
}
Key Takeaway: A subclass can access parent attributes directly only if they are not private. (e.g., givenName is accessible, but salary is private to Father).

Section 9: Introduction to Method Overriding

A subclass inherits all member variables and methods from its superclass. Overriding happens when a subclass provides its own specific version of an inherited method by using the exact same method name.

class Father {
    public void printSalary() {
        System.out.println("My salary is " + salary);
    }
}

class Son extends Father {
    // Overriding the parent's method!
    void printSalary() {
        System.out.println("I wish to earn 100k.");
    }
}
Key Takeaways:
  • Definition: Method overriding occurs when a subclass provides a specific implementation of a method already provided by its superclass.
  • Rule: To override, the method name must be exactly the same as the one in the superclass.
  • Purpose: It allows a subclass to provide specific behavior tailored to its needs, replacing inherited behavior.

Section 11: Refining Superclass Logic in Subclasses

Here is a practical example using a Circle superclass and a Cylinder subclass to calculate areas (and volumes).

class Circle {
    double radius;
    double getArea() {
        return (radius * radius * 22/7); 
    }
}

class Cylinder extends Circle {
    double height;
    
    // Overriding to calculate Cylinder volume
    double getArea() {
        return (radius * radius * 22/7 * height);
    }
}
Key Takeaways:
  • Practical Application: Inheritance is useful for modeling hierarchical relationships where subclasses refine the logic of superclasses.
  • Overriding: The Cylinder class overrides getArea() to include the height calculation, completely replacing the simple circle formula.

Section 12: Invoking Parent Methods via the Super Keyword

Instead of rewriting the entire logic in the overridden method (like repeating radius * radius * 22/7), we can access the original superclass method using the super keyword.

class Cylinder extends Circle {
    double height;
    
    double getArea() {
        // super.getArea() calls the Circle's getArea() method!
        return (super.getArea() * height);
    }
}
Execution code: 
Cylinder c1 = new Cylinder();
c1.radius = 10; c1.height = 100;
System.out.println(c1.getArea());
Output...
Key Takeaways:
  • The super keyword allows a subclass to invoke methods or constructors from its immediate parent class.
  • Use Case: This is essential when you want to reuse the base logic from the superclass method and extend it with additional logic in the subclass, rather than replacing it entirely.
