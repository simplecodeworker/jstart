What is the difference between a subclass and a superclass?

Correct Answer

Superclass (Parent/Base Class): The class whose properties and behaviors are inherited by another class. It is more generic and abstract.

Subclass (Child/Derived Class): The class that inherits from the superclass. It can add its own specific fields and methods, and modify (override) existing ones inherited from the parent.

OOP allows you to derive new classes from existing classes. What is this called?

Correct Answer

Inheritance (allowing the reusability of fields and methods of an existing class).

3

The statement super() or super (parameters) must appear in which line of the subclass constructor?

a. any line
b. 2nd line
c. 1st line
d. last line

Correct Answer

c. 1st line
In Java, if you explicitly call a superclass constructor with super(), it must be the very first statement inside the subclass constructor block.

What is it called when the subclass modifies the methods defined in the superclass?

Correct Answer

Method Overriding.

Which of the following is NOT true?

a. Every object is automatically an instance of Object class.
b. Every class in Java is descended from the Object class.
c. Object class is the root class of all Java class.
d. object strHello in statement String strHello = "Hello"; is an instance of String class, not Object class.

Correct Answer

d. object strHello in statement String strHello = "Hello"; is an instance of String class, not Object class.
This statement is NOT true because String is a subclass of Object. Therefore, any instance of String is implicitly also an instance of Object.

7

Analyze the following 2 classes:

ParentClass.java 
public class ParentClass {
    public double getResult(double i, double j) {
        return i * j;
    }
}
ChildClass.java 
public class ChildClass extends ParentClass {
    public double getResult(double m, double n) {
        return Math.pow(m, n);
    }

    public double doMutiplication(double i, double j) {
        return i * j;
    }

    public double getResult(double i, double j, double k) {
        return i * j * k;
    }

    public double getResult(int i, int j) {
        return i * j;
    }
}

In ChildClass, which method overrides following ParentClass method?

Correct Answer

The following method overrides the ParentClass method:

public double getResult(double m, double n)

Explanation: Overriding requires the exact same method signature (name, parameter types, and count). In Java, parameter names (like i, j vs m, n) do not matter for overriding. The other getResult methods in the child class are overload variants (different signatures due to differing argument lists or primitive types).
