Understanding CheckPalindrome.java

A beginner-friendly breakdown of how to check if a string reads the same forwards and backwards in Java.

Example

import java.util.Scanner;

// CheckPalindrome.java: Check whether a string is a palindrome
public class CheckPalindrome {
  /** Main method */
  public static void main(String[] args) {
    // Prompt the user to enter a string
    System.out.println("Please enter a string:"); 
    Scanner sc=  new Scanner(System.in);
    String s = sc.nextLine();
    // Declare and initialize output string
    String output = "";

    if (isPalindrome(s)) 
      output += s + " is a palindrome.";
    else 
      output += s + " is not a palindrome.";
    
    // Display the result
    System.out.println(output); 
    sc.close();
  }

  /** Check if a string is a palindrome */
  public static boolean isPalindrome(String s) {
    // The index of the first character in the string
    int low = 0;

    // The index of the last character in the string
    int high = s.length() - 1;

   /*String s1 = String.valueOf(3.4);
    System.out.println(s1);*/	
    while (low < high) {
      if (s.charAt(low) != s.charAt(high))
        return false; // Not a palindrome, return to main method, 
                       //skipping the rest statements in this method
        low++;//previous pair equals, continue to compare next pair 
        high--;
    }
	
    return true; // The string is a palindrome
  }
}

4. Setting Up the Pointers

This is where the actual logic happens. To check for a palindrome, we use a "two-pointer" approach. We create a low index pointing to the very first character (position 0), and a high index pointing to the very last character (length of string minus 1). 

    while (low < high) {
      if (s.charAt(low) != s.charAt(high))
        return false; // Not a palindrome, return immediately
        
      low++;  // previous pair equals, move forward
      high--; // previous pair equals, move backward
    }

  }

5. The Comparison Loop

A while loop runs as long as the low pointer hasn't crossed the high pointer. It checks if the characters at those two spots match.

If they don't match: It's definitely not a palindrome, so we immediately return false.
If they do match: We move the pointers closer to the middle (low++ and high--) and check the next pair.

If the loop finishes checking the whole string without failing, it returns true.
