Java String Comparison Explained

Understanding how Java handles Strings in memory is a crucial step for beginners. Let's break down the difference between == and .equals() step-by-step.

1. The String Pool vs. The Heap

In Java, there are two ways to create a String. When you use the new keyword (like s1, s2, s3), you are forcing Java to create a brand new object in the Heap memory every single time, even if the text is identical.

When you use double quotes directly (like s4, s5), Java checks a special area called the String Pool. If "welcome" already exists there, it just points the new variable to the existing word. This saves memory! Therefore, s4 and s5 point to the exact same location. 

    System.out.println(s1.equals(s2)); 
    // false, s1 and s2 have different contents
    
    System.out.println(s2.equals(s3)); 
    // true, s2 and s3 have the same contents
    
    System.out.println(s2==s3); 
    // false, s2 and s3 must have the same reference

2. Value vs. Memory Address

  • .equals() asks: "Do these strings contain the exact same characters, in the exact same case?"
    s1 ("Welcome" with a capital W) and s2 ("welcome") do not match. But s2 and s3 do!
  • == asks: "Do these variables point to the exact same physical memory address?"
    Because we used the new keyword for both s2 and s3, they are two separate objects living in different houses. So, == is false.
    System.out.println(s4==s5);
    // true, s4 and s5 have same reference and contents
    
    System.out.println(s4==s3);
    // false, s4 and s3 have different reference
    
    System.out.println(s4.equals(s3));
    // true, s4 and s3 have same content
    
    System.out.println(s1.equalsIgnoreCase(s5));
    // true

3. Proving the String Pool

Remember how s4 and s5 were created without the new keyword? Because they share the same spot in the String Pool, s4 == s5 is true!

However, s4 (in the pool) and s3 (in the heap) live in different places, making s4 == s3 false. They do still have the exact same text, which is why .equals() works perfectly between them. 

    s4="bye";

    s5=s4;
    // s4 and s5 do share the same memory location

    // now s4 and s5 share the same memory location

4. Reassignment & Memory Sharing

When we say s4 = "bye", Java doesn't change the word "welcome" into "bye" (Strings are immutable, meaning they cannot be changed). Instead, it creates a new "bye" in the pool and points s4 to it. s5 remains untouched, pointing at "welcome".

Finally, s5 = s4 literally tells Java: "Make s5 point to exactly the same memory address that s4 is pointing to." Because they now share an address again, s4 == s5 becomes true.
