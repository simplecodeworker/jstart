01. Initialization

Introduction to the String Class and Object Creation

🎯 Key Learnings & Takeaways

In Java, a String is not a simple primitive type (like int or char). Instead, it is an object instance of the class java.lang.String.

💡 Important Takeaway:
  • String Literals: Easiest syntax. Reuses pool data.
  • Constructor with new: Allocates a brand-new distinct block in memory.
  • Constructor with char[]: Converts character sequences to structured text.

Slide Concept Code

// Method 1: Using string literal assignment
String m1 = "Welcome to Java!";

// Method 2: Explicit Heap construction
String m2 = new String("Welcome to Java!");

// Method 3: Converting char array
char[] c = new char[] {'c','o','v','i','d'};
String m3 = new String(c); // "covid"

// Method 4: Constructor copying
String m4 = new String(m3);

02. Optimization

Understanding String Literals and Memory Optimization

🎯 Key Learnings & Takeaways

Java handles strings using a special optimization memory called the String Constant Pool (SCP) inside the Heap.

💡 Memory Pool Principle:
  • When a string literal is created, the JVM checks the Pool first.
  • If it already exists, the reference is reused. No duplicate objects are allocated.
  • Using the new keyword bypasses pool reuse, creating distinct separate objects.

String Constant Pool vs Heap Memory Layout

s1 = "Hi" → Registers "Hi" in Pool
s2 = "Hi" → Reuses Pool Reference
s3 = new String("Hi") → Creates NEW Object on general Heap

06. Critical Rules

Essential Characteristics: Immutability and Character Retrieval

🎯 Key Learnings & Takeaways

Two core rules to remember: 1. Strings are immutable. You cannot change a String's content. Any modification method (like `substring`) returns a new String instead of modifying the original. 2. No Bracket Access: Unlike C++ or Python, Java strings are objects, not simple primitive arrays. Access characters using .charAt(index), not brackets like s[index].

Immutability vs Mutation Flowchart

s1 = "text" ⟶ Points to "text" in pool
s1.substring(0,2) ⟶ Creates NEW separate String object "te". s1 remains unchanged!

07. Format Parsers

Converting Non-String Data Types to Strings

🎯 Key Learnings & Takeaways

To convert primitive types (like ints, doubles, and booleans) or arrays into a String object, use the static utility method String.valueOf(...).

💡 Method Insights:
  • It is a static class method, not an instance method.
  • It is overloaded to handle almost all primitive data types in Java.

Slide Static Overloads Code

String s2 = String.valueOf(23);     // "23"
String s3 = String.valueOf(56.78);  // "56.78"

09. Pointer evaluation

Reference Comparison vs. Content Comparison using ==

🎯 Key Learnings & Takeaways

The relational comparison operator == does not check the text characters inside String objects. Instead, it checks if they point to the exact same memory reference (address).

💡 Key Rule:
  • Two separate new String() instances always reside at different memory locations.
  • Therefore, s1 == s2 evaluates to false, even if they contain the exact same characters.

Memory Allocation Reference Layout

s1 = new String("Hi") Reference: Heap Address A
s2 = new String("Hi") Reference: Heap Address B
s1 == s2 FALSE (Addresses Differ)

10. True Equality

Value Evaluation using the equals() Method

🎯 Key Learnings & Takeaways

To compare the actual character content inside two Strings, you should use the .equals(...) method instead of `==`.

💡 Core Takeaways:
  • The equals() method compares character sequences index-by-index.
  • This method is case-sensitive. For case-insensitive comparisons, use .equalsIgnoreCase(...).

Reference vs Value Equality comparison

String s1 = new String("Java");
String s2 = new String("Java");
s1 == s2 false
s1.equals(s2) true
