SECTION 3 & 4

Syntax: Declaration & 'new'

Declaring is telling Java about the variable. Creating is asking for memory.

int[] marks; // Declaration
marks = new int[5]; // Creation: allocate 5 slots
System.out.println(marks.length); // Prints 5

Declaration

The brackets [] indicate it's an array. Use type[] name.

Creation ('new')

The new keyword tells Java to reserve actual space in RAM.

Takeaway: Declaration defines the variable; 'new' allocates memory. The size is fixed at the moment of creation.

SECTION 10

Troubleshooting Errors

Error: Missing Size

int[] data = new int[]; 
// WRONG: Size missing

Error: Broken Shorthand

int[] data;
data = {1, 2, 3}; 
// WRONG: 'new' required here
// data = new int[] {1, 2, 3};

Takeaway: Size is mandatory at creation. Shorthand {...} only works if declared and assigned on the same line.

SECTION 11

Three Ways to the Same Goal

Depending on when you know your data, Java provides three different paths to create the exact same array in memory. All three of these are 100% correct!

A

The "Instant" Array Literal

int[] data = {1, 2, 3};

Best when you know the values immediately at the moment of declaration.

B

The "Delayed" Initialization

int[] data; 
data = new int[] {1, 2, 3};

Essential if you declare the variable first and provide values later. Note: the new int[] is mandatory here!

C

The "Manual" Assembly

int[] data; 
data = new int[3]; 
data[0] = 1; 
data[1] = 2; 
data[2] = 3;

Best when you know the size, but the values come from calculations or user input later.

Takeaway: All three methods create an array of size 3 containing the same values. Choosing between them is simply a matter of timing and coding style.
