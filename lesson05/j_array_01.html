Array Access & Initialization Syntax

Arrays are objects that store multiple values of the same type. First, declare a reference variable, then use the new keyword to allocate memory.

int[] marks;           // Declare
marks = new int[5];    // Instantiate 5 slots
// Access: arrayName[index]

Section 2

Zero-Based Indexing & Boundary Rules

Java arrays use zero-based indexing. The first slot is always index 0, and the last slot is size - 1.

int[] marks = new int[5];
marks[0] // 1st element
marks[1] // 2nd element
marks[4] // 5th (Last) element
marks[5] // ERROR! (Out of Bounds)
idx 0
idx 1
idx 2
idx 3
idx 4
INVALID

Accessing index 5 on size 5 will crash your program!

Takeaway 🔑

Always remember: "Start at zero, end at size minus one."

Section 4

Automating Array Processing with Loops

Manual assignment is slow. Use loops combined with the .length property to process an array of any size automatically.

for (int k = 0; k < marks.length; k++) {
    marks[k] = 10 + k;
}
The .length Property

In Java, arrayName.length returns the total number of slots. It is used in the loop condition to prevent "out of bounds" errors.

Automation Logic

Start: 0
Condition: < length
Increment: k++

Takeaway 🔑

Don't manually assign values to 100 elements; use a loop and the .length property to do the work for you.

Section 6

Array Pass-By-Reference in Methods

When you pass an array to a method, you are passing a reference. Changes made inside the method happen to the actual original array.

int[] mark = {30, 40, 50};
changeMark(mark, 5); // Call
// Original becomes {35, 45, 55}

public static void changeMark(int[] array, int val) {
    for(int i=0; i<array.length; i++)
        array[i] += val;
}

Reference Concept

The method receives the address of the original object, not a copy.

Takeaway 🔑

Be careful when passing arrays to methods; if the method changes an element, the original array in your main program changes too!

Section 7

The Enhanced For Loop (For-Each)

The "Enhanced for loop" (or for-each) is a simplified syntax for visiting every element without needing an index counter i.

Enhanced Syntax

for (double num : numbers) {
    System.out.println(num);
}

Equivalent Logic

for (int i=0; i<len; i++) {
  double num = numbers[i];
  System.out.println(num);
}

Takeaway 🔑

Use the enhanced for loop when you want to look at every item in an array without worrying about index numbers.

Section 8

Variable Number of Arguments (Varargs)

Varargs allow a method to accept any number of inputs of the same type. Inside the method, Java treats these inputs as a standard array.

static int add(int ... numbers) {
    int sum = 0;
    for(int n : numbers) sum += n;
    return sum;
}
// Called as: add(1, 2) or add(1, 2, 3, 4)
Flexibility

No need to create multiple overloaded methods for different numbers of inputs.

Implementation

The ... syntax tells Java to bundle all trailing arguments into an array.

Takeaway 🔑

Varargs make your methods more flexible by allowing the caller to decide how many items to send.

Final Section

Size Persistence and Self-Assessment

Arrays have a fixed size. Once created, they cannot grow or shrink. To change size, you must create a new array object entirely.

Precision Syntax Checklist:
  • .length is a field (no parentheses). .length() is for Strings.
  • Index Limits: Last index is always length - 1. Accessing [length] is a mistake.

Takeaway 🔑

Precision in syntax matters—always mind the zero-based index and remember that array sizes are immutable once instantiated.
