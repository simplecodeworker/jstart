Section 01

The Syntax of Catching Exceptions

The Core Structure

Handling exceptions in Java involves three primary keywords that act as a safety net for your code.

  • try: Wraps the "dangerous" code that might fail.
  • catch: Specific blocks to handle various types of errors.
  • finally: The cleanup crew. Runs no matter what happens.
Mandatory: Curly braces {} are required even for single-line blocks! 
try {
    // Code that might throw exceptions
    openFile("data.txt");
}
catch (FileNotFoundException e) {
    // Handle specific error
}
catch (Exception e) {
    // Catch-all for other errors
}
finally {
    // Always runs (e.g., closing file)
}
Section Takeaway

Exceptions are managed by "trying" code and "catching" specific errors to prevent program crashes.

Section 02

Essential Rules

"Try is not alone, Catch is conditional, Finally is guaranteed."
1

Checked exceptions must be placed in a try block or declared.

2

Try cannot be alone; it needs at least one catch or finally.

3

If an exception hits, the remaining code in the try block is skipped.

4

Finally runs even if there is a return or exit statement before it.

5

Catch only executes if the exception type matches the error thrown.

Section Takeaway

You can't have a try without a partner (catch or finally), and finally is your "safety net" that always runs.

Section 05

Accessing Exception Details

e.getMessage()

Returns a short, descriptive string explaining the core reason for the failure (e.g., "/ by zero").

e.toString()

Returns the full class name of the exception plus the message. Better for identifying the error type.

e.printStackTrace()

The debugger's best friend. Lists every method call that led to the crash, including line numbers.

Section Takeaway

The exception object 'e' contains useful data that helps developers debug exactly where and why a crash happened.

Section 06

Pros, Cons, and Resources

Advantages

  • Robustness: Prevents abnormal termination of the software.
  • Readability: Separates error logic from main business logic.

Disadvantages

  • Overhead: Requires more CPU resources to process exceptions.
  • Verbosity: Can make simple code look bulky if overused.
