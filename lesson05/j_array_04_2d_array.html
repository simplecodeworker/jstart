1. Introduction to 2D Arrays

Think of a 2D array as a Table. It is essentially an "array of arrays" where data is organized into rows and columns.

Syntax & Declaration

// Declaring a 2-D Array
int[][] arrayName;

// Create (4 rows, 3 columns)
arrayName = new int[4][3];

// Literal Initialization
int[][] array = {
  {1,2,3}, /* row 0 */
  {4,5,6}, /* row 1 */
  {7,8,9}, /* row 2 */
  {10,11,12} /* row 3 */
};

2. Manual Initialization and Indexing

Indices in Java always start at 0. To access an element, use array[row][column].

Interactive Index Explorer

Access Code:

array[0][0]

Click a cell to see its Java index.

int[][] array = new int[4][3];
array[0][0] = 1;
array[3][2] = 12; // The very last element

Takeaway: Manual assignment is essential when data is calculated dynamically or when you don't know the values at the time of creation.

3. The Length Property

The .length property behaves differently depending on where you call it.

Rows

arrayName.length

Returns the number of rows in the 2D array.

Columns

arrayName[i].length

Returns the number of columns (size) of row i.

Takeaway: Always use .length in loops! Hardcoding numbers leads to ArrayIndexOutOfBoundsException, especially in Ragged Arrays.

4. Creating Ragged Arrays

Java allows "Ragged" (or jagged) arrays where each row can have a different number of columns.

// Step 1: Define rows ONLY
int[][] arr = new int[3][];

// Step 2: Define each row separately
arr[0] = new int[2]; // row 0 has 2 cols
arr[1] = new int[4]; // row 1 has 4 cols
arr[2] = new int[1]; // row 2 has 1 col

Scenario: Months have different numbers of days. A ragged array days[12][] saves memory by only allocating what is needed for each month.

Ragged Array Memory Layout

Takeaway: Ragged arrays are memory-efficient when your data sets vary significantly in size (like days in a month or words in a sentence).

Quick Check

What is the correct syntax to start a ragged array with 3 rows?

new int[3][]
new int[3][3]

5. Array References and Memory

In Java, array variables are references. Assigning one array to another doesn't copy the data—it copies the pointer!

Takeaway: Modifying an array via one reference affects all other references to it. Arrays are objects, and variables hold their addresses.
