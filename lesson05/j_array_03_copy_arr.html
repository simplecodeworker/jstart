Section 1

Manual Element-by-Element Copying

The most fundamental method involves using a for loop to visit every index. You manually take a value from the source and place it into the target. 

for(int i=0; i < sourceArray.length; i++){
    targetArray[i] = sourceArray[i];
}

Key Takeaway

Full control over the process. Requires manual instantiation of the target array first. The arrays remain independent.

Section 2

High-Performance: System.arraycopy

Java provides a native method for copying. It is faster for large arrays and allows copying specific segments. 

System.arraycopy(
  sourceArray,  // From where?
  src_pos,      // Starting at?
  targetArray,  // To where?
  dst_pos,      // Ending at?
  nElements     // How many?
);

Key Takeaway

Best for large arrays and partial copies. Highly efficient because it uses low-level system calls.

Section 3

Concise Duplication: .clone()

Need a total copy in one line? The clone() method creates a brand new object and copies everything for you. 

targetArray = sourceArray.clone();

Key Takeaway

Simplest syntax. Automatically creates the target array object. Performs a "shallow" copy.

Section 4

The Danger Zone: Reference Assignment

Warning: Using the equals sign (=) is NOT copying. 

targetArray = sourceArray; // ❌ WRONG

This creates a Reference Copy. You now have two variables pointing to the exact same physical spot in memory.

Key Takeaway

Changes made to targetArray will change sourceArray! The original target data is lost and deleted by Garbage Collection.
