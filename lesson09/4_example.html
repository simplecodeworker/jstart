Understanding Java Polymorphism

A visual guide to object-oriented magic for beginners.

What is Polymorphism?

The word "polymorphism" means "many forms". In Java, it allows us to perform a single action in different ways. Think of it like a remote control: the "Play" button does one thing for a TV, but something different for a DVD player.

In this example, we'll see how a single Animal variable can take the form of a Cow, a Cat, or a Snake, and how each responds differently when asked to talk().

Animal.java 
// Animal is superclass of Cow, Cat and Snake subclasses
package oop7TestAnimalPolymorphsim;

class Animal {
    private int legs = 4;
    
    public void talk() {
        // If subclass doesn't have talk() method, this method is called.
        System.out.println("Animals can't talk");
    }
}
The Blueprint

The Base Class

Think of Animal as a generic category. It provides the basic features that all animals share.

  • It defines a default talk() behavior. If a specific animal doesn't know how to talk, it will fall back to saying "Animals can't talk".
  • This is our Superclass (or parent class). Others will inherit from it.
Cat.java 
// Cat is subclass with own talk() method
package oop7TestAnimalPolymorphsim;

class Cat extends Animal {
    public void talk() {
        System.out.println("Meow!");
    }
}
The Override

The Cat Class

The Cat class extends Animal. This means a Cat is an Animal.

  • Instead of using the generic Animal talk, the Cat provides its own specific version of the talk() method.
  • This is called Method Overriding. The Cat is changing the behavior it inherited from its parent.
Cow.java 
// Cow is subclass with own talk() method
package oop7TestAnimalPolymorphsim;

class Cow extends Animal {
    public void talk() {
        System.out.println("Moo");
    }
}
Another Variation

The Cow Class

Similar to the Cat, the Cow class also extends Animal.

  • It defines its own unique response when asked to talk(): "Moo".
  • We now have two distinct shapes (forms) an Animal can take, each with custom behavior.
Snake.java 
// Snake is subclass with no talk() method, 
// hence uses Animal superclass talk() method
package oop7TestAnimalPolymorphsim;

class Snake extends Animal {
    // only have its own data, will inherit 
    // parent class talk() method
    private int legs = 0;
}
Inheritance

The Snake Class

The Snake class is interesting. It extends Animal to redefine its specific attributes (like having 0 legs).

  • Notice it does not have a talk() method of its own.
  • Because it didn't override the method, it automatically inherits the default behavior from the Animal class.
TestAnimals.java The Magic Happens Here 
package oop7TestAnimalPolymorphsim;

public class TestAnimals {
  public static void main(String[] args) {

      // 1. Create specific subclass objects
      Cat babe = new Cat();
      Cow cutie = new Cow();
      Snake glass = new Snake();

      // 2. Create a superclass reference
      Animal wild = new Animal();

      // 3. Polymorphism in action: Cow
      wild = cutie; // Animal behaving like Cow
      wild.talk();  // Prints: "Moo"

      // 4. Polymorphism in action: Cat
      wild = babe;  // Animal behaving like Cat
      wild.talk();  // Prints: "Meow!"

      // 5. Polymorphism with Inheritance: Snake
      wild = glass; // Animal behaving like Snake
      wild.talk();  // Prints: "Animals can't talk"

  }
}

Putting it all together

This is where polymorphism shines. We use a single reference variable (wild) to hold different types of objects and execute their specific behaviors.

Step 3: The Cow Form

We point the generic wild reference to the cutie Cow object. When we call wild.talk(), Java knows it's currently pointing to a Cow, so it uses the Cow's overridden method.

Step 4: The Cat Form

We reassign the same generic wild reference to the babe Cat object. Now, calling the exact same command wild.talk() produces a different result ("Meow!") because the underlying object shape has changed.

Step 5: The Snake Form

Finally, we point it to the glass Snake object. We call wild.talk() again. Since Snake didn't provide its own specific talk method, Java climbs up the family tree and executes the default method found in the Animal class.

The Core Lesson

The method that gets executed is determined by the actual object type (Cat, Cow, Snake) created in memory using `new`, not by the reference type (Animal) holding it. This is Dynamic Method Dispatch in action!
