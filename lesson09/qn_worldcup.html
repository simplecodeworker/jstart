Inheritance Question: World Cup Tracker

You are building a tournament tracking application. The system manages general information about a sports team and adds specific tournament-tracking data for national teams participating in the ongoing FIFA World Cup.

Create a new project with 3 Java source files: a superclass file, a subclass file, and a test file using the specifications and names below.

UML Diagram Specification

SUPERCLASS
Team
- countryName: String
+ Team()
+ Team(String)
+ getCountryName(): String
+ setCountryName(String): void
+ describe(): void
Extends
SUBCLASS
WorldCupTeam
- tournamentStage: String
+ WorldCupTeam()
+ WorldCupTeam(String, String)
+ getTournamentStage(): String
+ setTournamentStage(String): void
+ describe(): void

Task Checklist

Superclass
  • Private string countryName.
  • No-argument constructor defaults to "Unassigned Country".
  • Parameterized constructor accepts a String.
  • Getters and setters implemented.
  • Method describe() displays country.
Subclass
  • Private string tournamentStage.
  • No-arg constructor chains to super and defaults to "Not Started".
  • Parameterized constructor chains to super using super(countryName).
  • Getters and setters for stage.
  • Overridden describe() runs super.describe() first, then appends stage details.
Driver
  • Instantiate two WorldCupTeam objects (one default, one parameterized).
  • Print initial properties of default object using describe().
  • Modify properties using setter mutator methods.
  • Describe the object once more to confirm updates are active.

END OF QUESTION
