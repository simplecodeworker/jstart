Java Inheritance & Polymorphism

Solution guide for the Employee and SalesMan class implementation.

Original Problem Context

7.2 The following is an Employee class. All employees have a name and a serial number, and each employee earns a weekly salary. But the weekly salary varies with the types of the employee.

File: Employee.java (Provided) 
//The Employee class is a base class. It is intended to be
//extended and each of its subclasses need to override the
//calcWeeklySalary() method based of the type of employee.
 
public class Employee {
    String name, sSN;
    float weeklySalary;
 
    public Employee(){
        name = "";
        sSN = "";
    }
 
    public Employee(String name, String sSN){
        this.name = name;
        this.sSN = sSN;
    }
 
    public String getName(){
        return name;
    }
 
    public float getWeeklySalary() { 
        return weeklySalary; 
    }
 
    //The salary calculation is different for each type of employee
    //Here is a basic weeklySalary.
    public void calcWeeklySalary(){
        weeklySalary = 15 * 5; //take ITP student as an example
    }
}

For salesmen, they are paid a flat weekly rate plus a commission based on their total sales for the week: weeklySalary = baseWkPay + totalWkSales*commission. The following SalesMan class is a child class of Employee class, please override method calcWeeklySalary() to calculate the weekly salary for a salesman.

File: SalesMan.java (Skeleton Provided) 
//A sales person's weekly salary is based on a flat weekly
//rate plus a commission from the total sales.
 
public class SalesMan extends Employee{
    String region;
    private float baseWkPay, totalWkSales, commission;
 
    public SalesMan(){ //no arg constructor to initialize basic info
        totalWkSales = 0.0f;
        commission = 0.10f;           //base rate is 10%
        baseWkPay = 250.0f;           //base pay is $250/week
    }
 
    public SalesMan(String name, String sSN, String reg, float basePay, float salesTotal, float comm){
        /*another constructor takes in parameters to initialize more variables*/
        super(name, sSN);
        region = reg;
        totalWkSales = salesTotal;
        commission = comm;
        baseWkPay = basePay;
    }
 
    public void calcWeeklySalary(){
        /*Add your own codes here to override employee’s calcweeklySalary here*/
    }
}

Assignment Tasks:

  • Please override method calcWeeklySalary() to calculate the weekly salary for a salesman.
  • Write a Java Application to create 2 SalesMan objects using different constructors, calculate and print out their weekly salary.

1 Overriding the Calculation Method

Correct Answer (Inside SalesMan.java) 

    @Override
    public void calcWeeklySalary() {
        // weeklySalary is inherited from the Employee class.
        // We override the parent method to apply the SalesMan specific formula.
        weeklySalary = baseWkPay + (totalWkSales * commission);
    }

Beginner Watch-outs

  • The @Override annotation: While not strictly mandatory, it is highly recommended. It tells the compiler you intend to override a method. If you make a typo (e.g., calcWeeklySalry()), the compiler will catch it and throw an error.
  • Inherited Variables: Notice how we can use weeklySalary directly without declaring it? That's because it was declared as a package-private variable (float weeklySalary;) in the parent Employee class.
  • Return Type: The parent class defined this method as public void calcWeeklySalary(). You cannot change the return type to float in the child class; it must match exactly.

2 Writing the Java Application

Correct Answer (SalesApp.java) 

public class SalesApp {
    public static void main(String[] args) {
        
        // 1. Using the no-argument constructor
        // Default values: basePay=250.0f, sales=0.0f, comm=0.10f
        SalesMan sm1 = new SalesMan();
        
        // IMPORTANT: Calculate the salary before trying to print it!
        sm1.calcWeeklySalary();
        System.out.println("Default SalesMan Salary: $" + sm1.getWeeklySalary());

        // 2. Using the parameterized constructor
        // Parameters: (String name, String sSN, String reg, float basePay, float salesTotal, float comm)
        SalesMan sm2 = new SalesMan("Alice Smith", "987654321", "East", 300.0f, 5000.0f, 0.15f);
        
        // Calculate the salary for the second object
        sm2.calcWeeklySalary();
        System.out.println(sm2.getName() + "'s Salary: $" + sm2.getWeeklySalary());
    }
}

Beginner Watch-outs

  • Order of Execution Matters: You must call calcWeeklySalary() before you call getWeeklySalary(). If you forget to calculate it first, the inherited weeklySalary float variable will default to 0.0!
  • The f suffix for Floats: Notice the numbers passed into the constructor (e.g., 300.0f). In Java, a decimal number like 300.0 is treated as a double by default. Leaving off the 'f' will cause a compile-time error.
  • Constructors & super(): When new SalesMan(...) is called, it calls super(name, sSN);. This runs the Employee class constructor to handle the name and SSN setup automatically.
