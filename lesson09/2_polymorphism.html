Theoretical Concept of Polymorphism

The word Polymorphism comes from Greek, meaning "many forms".

"An object of a subclass can be used by any code designed to work with an object of its superclass."

Why is this logical? Because a subclass inherits everything from its superclass. Therefore, the subclass is guaranteed to be able to do everything that the superclass was designed to do.

The Substitutability Rule

Expects: Animal
Cat Object
Snake Object

Both Cat and Snake fit perfectly into a slot designed for an Animal.

Key Takeaway

The core rule is substitutability—wherever a superclass is expected, a subclass can be safely used because it has at least the same behaviors.

Implementing Method Overriding

Let's look at our first code example. We define the generic Animal and specific subclasses that override the talk() method. 

class Animal {
  void talk() {
    System.out.println("Not sure how."); 
  }
}

class Cat extends Animal {
  void talk() {
    System.out.println("Meow, meow!"); 
  }
}

class Snake extends Animal {
  void talk() {
    System.out.println("Hiss, hiss!"); 
  }
}

Dynamic Method Dispatch & Variable Assignment

Now we see polymorphism in action. A single reference variable of type Animal can point to different subclass objects dynamically at runtime.

Code Execution Visualizer 
public class testPolymorphism {
  public static void main(String[] args) {

    Animal a = new Animal();
    a.talk();


    Cat c = new Cat();
    Snake s = new Snake();


    a = c; // Reassignment
    a.talk();


    a = s; // Reassignment
    a.talk();
  }
}

Key Takeaway

A reference variable of a superclass type (Animal a) can hold object instances of its subclasses. Java determines at runtime which version of the overridden method to execute based on the actual object assigned to it. This is Dynamic Method Dispatch.

Inheritance with Constructors & Encapsulation

In our second example, we introduce a superclass Vehicle with a private property. Notice how the subclass interacts with it. 

class Vehicle {
  private int speed;
  
  public Vehicle () { speed=50; }
  
  public void setSpeed (int s) { speed = s; }
  public int getSpeed () { return (speed); }
}

class SportsCar extends Vehicle {
  public String brand;
  
  public SportsCar(String b){ 
    brand = b; 
    setSpeed(100); 
  }
}
Vehicle
- speed : int (private)
+ Vehicle()
+ setSpeed(s: int)
+ getSpeed(): int
SportsCar
+ brand : String
+ SportsCar(b: String)

SportsCar cannot access speed directly, but uses inherited public methods.

Key Takeaway

The subclass SportsCar inherits public methods (setSpeed) from Vehicle, allowing it to safely modify private instance variables inherited from the parent class without violating encapsulation.

Polymorphism in Method Parameters

One of the most powerful uses of polymorphism is writing scalable methods. A method that accepts a superclass object as a parameter can accept any subclass of that object. 

public class testPolymorphism {
  public static void main(String[] args) {
    SportsCar sc1 = new SportsCar("BMW");
    
    System.out.println(sc1.getSpeed()); // Outputs 100
    
    // Passing SportsCar to a method expecting Vehicle
    stopVehicle(sc1); 
    
    System.out.println(sc1.getSpeed()); // Outputs 0
  }

  // Notice the parameter is type Vehicle
  public static void stopVehicle (Vehicle v) {
    v.setSpeed(0);
  }
}

Methods written to accept a superclass parameter can accept arguments of any subclass type. This makes your code highly reusable; you do not need to write separate stopVehicle methods for every distinct type of vehicle.
