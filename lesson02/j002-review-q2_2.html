Temperature Conversion in Java

Mastering basic I/O and arithmetic precision.

01 The Problem

Convert a user-provided Fahrenheit temperature into Celsius using the formula:

celsius = (5/9) * (fahrenheit - 32)

Your program reads a Fahrenheit degree in double from the keyboard, then converts it to Celsius and displays the result.
Hint: In Java, 5/9 is 0, so you need to use 5.0/9 in the program to obtain the correct result.

02 Common Beginner Mistakes

Integer Division Trap

In Java, 5 / 9 equals 0 because both are integers. You must use 5.0 / 9 to trigger floating-point division.

Did Not Close Scanner

Failing to call input.close() at the end of your program. While small programs might survive, this causes "resource leaks" in real-world applications.

Strict Input Types

Using nextInt() when the user might enter 98.6. This triggers a Runtime Error. Always use nextDouble() for measurements.

Unrequired Imports

Importing java.util.* unnecessarily or importing classes you don't use. Import only what you need (e.g., java.util.Scanner) to keep code clean.

The Final Java Code

import java.util.Scanner;


public class TempConverter {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
       
        System.out.print("Enter Fahrenheit: ");
       
        // Use nextDouble() to support decimals
        double fahrenheit = input.nextDouble();
       
        // 5.0 ensures floating point math
        double celsius = (5.0 / 9) * (fahrenheit - 32);
       
        System.out.println(fahrenheit + "F is " + celsius + "C");
       
        // Manually close the scanner to release resources
        input.close();
    }
}
