Part (a) Create Superclass (Team.java)

Question Requirements

  • Private string countryName.
  • No-argument constructor defaults to "Unassigned Country".
  • Parameterized constructor accepts a String.
  • Getters and setters implemented.
  • Method describe() displays country.
// File: Team.java

public class Team 
{
    // 1. Private string countryName.
    private String countryName;

    // 2. No-argument constructor defaults to "Unassigned Country".
    public Team() 
    {
        countryName = "Unassigned Country";
    }

    // 3. Parameterized constructor accepts a String.
    public Team(String countryName) 
    {
        this.countryName = countryName;
    }

    // 4. Getters and setters implemented.
    public String getCountryName() 
    {
        return countryName;
    }

    public void setCountryName(String countryName) 
    {
        this.countryName = countryName;
    }

    // 5. Method describe() displays country.
    public void describe() 
    {
        System.out.println("Country: " + countryName);
    }
}

Beginner's Watch-Out

Notice how the constructors do not have a return type (not even void) and match the exact name of the class. Also, in the parameterized constructor, using the this. keyword (this.countryName = countryName;) is crucial to differentiate the instance variable from the parameter passed in.

Part (b) Create Subclass (WorldCupTeam.java)

Question Requirements

  • Private string tournamentStage.
  • No-arg constructor chains to super and defaults to "Not Started".
  • Parameterized constructor chains to super using super(countryName).
  • Getters and setters for stage.
  • Overridden describe() runs super.describe() first, then appends stage details.
// File: WorldCupTeam.java

public class WorldCupTeam extends Team 
{
    // 1. Private string tournamentStage.
    private String tournamentStage;

    // 2. No-arg constructor chains to super and defaults to "Not Started".
    public WorldCupTeam() 
    {
    // In WorldCupTeam default constructor, calling super() is technically optional because Java automatically calls the no-arg constructor of the superclass. 
    // However, explicitly writing it is considered a good practice for readability and intentionality.
        super();
        tournamentStage = "Not Started";
    }

    // 3. Parameterized constructor chains to super using super(countryName).
    public WorldCupTeam(String countryName, String tournamentStage) 
    {
        super(countryName);
        this.tournamentStage = tournamentStage;
    }

    // 4. Getters and setters for stage.
    public String getTournamentStage() 
    {
        return tournamentStage;
    }

    public void setTournamentStage(String tournamentStage) 
    {
        this.tournamentStage = tournamentStage;
    }

    // 5. Overridden describe() runs super.describe() first, then appends stage details.
    @Override
    public void describe() 
    {
        super.describe();
        System.out.println("Tournament Stage: " + tournamentStage);
    }
}

Beginner's Watch-Out

The call to super() must be the very first statement inside a constructor. You cannot write any code before it! Additionally, observe the @Override annotation before the describe() method—it asks the Java compiler to verify that you are correctly replacing a method from the parent class, preventing silent typo bugs.

Part (c) Create Test Driver (TestTournament.java)

Question Requirements

  • Instantiate two WorldCupTeam objects (one default, one parameterized).
  • Print initial properties of default object using describe().
  • Modify properties using setter mutator methods.
  • Describe the object once more to confirm updates are active.
// File: TestTournament.java

public class TestTournament 
{

    public static void main(String[] args) 
    {
    // 1. Instantiate two WorldCupTeam objects (one default, one parameterized).
        WorldCupTeam team1 = new WorldCupTeam();

        WorldCupTeam team2 = new WorldCupTeam("Brazil", "Quarter Finals");

    // 2. Print initial properties of default object using describe().
        System.out.println("=== Default Object ===");
        team1.describe();

    // 3. Modify properties using setter mutator methods.
        team1.setCountryName("Japan");
        team1.setTournamentStage("Round of 16");

    // 4. Describe the object once more to confirm updates are active.
        System.out.println("\n=== Updated Default Object ===");
        team1.describe();

        System.out.println("\n=== Parameterized Object ===");
        team2.describe();
    }
}

Beginner's Watch-Out

Remember that you must use the new keyword to instantiate an object before calling methods on it. If you only type WorldCupTeam team1; without assigning it a new WorldCupTeam(), the variable is null and will crash with a NullPointerException if you try to use it! Also, the main method is strictly required to run your Java program.
