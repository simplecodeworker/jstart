Introduction & Objectives

Methods are the foundation of modular programming. Instead of writing one giant wall of code, we break complex tasks into smaller, manageable, and reusable blocks.

2

Understanding Methods

"A method resides inside a class and represents a collection of statements that execute together to perform an operation."

Tidy Code

Organizes code into logical units, making it easier to read and debug.

Reusability

Write once, call many times. Reduces redundancy across your program.

Encapsulation

Hides complex internal logic. Users only need to know how to call the method.

3

Structure & Syntax

MethodExample.java
public static int max(int num1, int num2) // Header
{ // Body starts
int result = 0;
if (num1 > num2)
result = num1;
else
result = num2;
return result;
}

Knowledge Drop: Signatures

A Method Signature consists ONLY of:

methodName (formalParameterList)

Note: This does not include modifiers or return types!

The Void Rule

If a method performs an action (like printing) but gives nothing back to the caller, use the keyword void.

public static void sayHello() { ... }
