Section 9

Method Overloading

Method overloading allows multiple methods to share the same name but have a different signature. There cannot be two methods with the same name and signature.

What Counts in method signature?

  • Different number of parameters
  • Different types of parameters

What DOESN'T count in method signature?

  • Changing just the Return Type
  • Changing just Modifiers (public/static)

Code Examples Breakdown

INVALID 
public static int max(int n1, int n2) { ... }
public static float max(int n1, int n2) { ... }

Error: Signatures (int, int) are identical! Return type doesn't matter. There cannot be two methods with the same name and signature.

VALID 
public static int max(int n1, int n2) { ... }
public static int max(int n1, int n2, int n3) { ... }

Success: Different parameter counts make unique signatures.

Section 10

Scope of Variables

Scope defines where a variable is "alive" and accessible. In Java, local variables are bound by their containing { brackets }. Scope is from its declaration to the end of the block containing the variable. Outside the scope, it is invisible.

Visibility

Variables declared inside a method (like max) aren't visible to main.

Variable Hiding

Outer field variables are hidden when a local variable with the same name is declared within a method or inner block. However, Java does not allow local variables to be redeclared with the same name in overlapping scopes within the same method; attempting to do so results in a compilation error.

Formal Parameters

Method’s formal parameters cannot be re-declared in the method body. Its scope covers the entire method..

Scope Life-Cycle Interactive

public static void main(...) {
int x = 5;
if(true) {
int y = 10;
// y is alive here
}
// y is DEAD here!

Practical Scope Examples

1. Method Isolation 
public static void main(String[] args) {
    int x = 10;
    anotherMethod();
    // System.out.println(y); // ERROR!
}

public static void anotherMethod() {
    int y = 20; 
    // y is only visible here
}

Variables like y cannot be accessed outside the method they were declared in.

2. Variable Hiding (Nested)

ERROR

public static void main(String[] args) {
    int i = 1;
    for (int k = 0; k < 5; k++) {
        // int i = 2; // COMPILER ERROR
        // Cannot re-declare 'i' in same scope
    }
}

You cannot declare a variable with the same name if it's already "visible" in an outer block.

ALLOWED

class Test {
    int y = 5; // outer field

    void method() {
        if (true) {
            int y = 10; // hides the field y within this block
            System.out.println("Inner block y = " + y); // 10
        }

        System.out.println("Field y = " + this.y); // 5
    }

    public static void main(String[] args) {
        Test t = new Test();
        t.method();
    }
}

You can declare a local variable within a method or inner block using the same name as a field, which hides the outer field variable within that scope.

Important Rule: Local variables must be initialized before use. If you declare int x; and try to print it without giving it a value, Java will throw a compilation error.

System.out & Overloading

Did you know?

`System.out.println()` is actually a collection of many overloaded methods in the `PrintStream` class!

println(int i)
println(double d)
println(String s)
println(Object obj)

String Concatenation Trap

Code:
"I got " + 9*10+9 + "."
Output:
"I got 909."
Code:
"I got " + (9*10+9) + "."
Output:
"I got 99."

* Multiplication happens before addition. Concatenation happens left-to-right unless parentheses are used.
