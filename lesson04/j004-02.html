Method Invocation & Data Flow

The Concept

Calling a method (invocation) triggers a jump in execution. Think of it as pausing your current task to let someone else handle a specific job, then waiting for them to bring back the result.

IMPORTANT NOTES

  • Invocation: k = max(i, j) triggers the method.
  • Parameters: Arguments must match formal parameters in terms of number of parameters, data type and order.
  • Data Type Consistency: The returned result type MUST match the returnValueType in the header.
Main.java 
public static void main(String[] args) {
  int i = 5;
  int j = 2;
  int k = max(i, j); // Invocation

  System.out.println("Max is " + k);
}

public static int max(int num1, int num2) {
  int result = (num1 > num2) ? num1 : num2;
  return result; // Must be int!
}

Signature Matching Rules

Communication between call and definition must be precise. Match must be consistent in: 1. Number, 2. Data Types, and 3. Order.

8

Practical Rules & Pitfalls

Rule 1: Calling Syntax

CORRECT int k = max(i, 2);
WRONG int k = max(int i, int j);

Don't include data types in the call.

Rule 2: Arguments Must Match Parameters

Arguments must match parameters in terms of number of parameters, types and order.

CORRECT int k = max(i, 2);
WRONG int k = max(i, j, k);

Must match number of parameters

WRONG int k = max(2.0, 3.0);

Must match parameter type

Rule 3: Using the Result

If a method returns a value, capture it or use it! Otherwise, it is "lost" in memory.

// Captured
int k = max(5, 10);
// Result is lost!
max(5, 10);

Rule 4: Void Methods

Void methods do not return data. They perform an action (like printing).

  • Cannot be assigned to a variable.
  • return; is allowed to exit early.
  • return 0; is a compilation error.

Rule 5: Pass-by-Value

Java passes copies of values. If max changes num1, the original i in main remains unchanged.
