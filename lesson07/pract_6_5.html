Q6.5.1: Convertor Class

Design a Convertor class with static and non-static methods for temperature conversion.

public class Convertor {
    // (a) Static method: belongs to the class itself
    public static double cToF(double c) {
        // WARNING: Use 9.0/5.0 instead of 9/5. 
        // Integer division 9/5 would result in 1, not 1.8!
        return (9.0 / 5.0) * c + 32;
    }

    // (b) Non-static method: belongs to an instance of the class
    public double fToC(double f) {
        return (f - 32) * 5.0 / 9.0;
    }
}

Beginner Watch-out:

When performing division in Java, 9 / 5 performs integer division (resulting in 1). Always ensure at least one operand is a double (9.0 / 5.0) to get the floating-point result.

Q6.5.2: Testing Class

Design a testing class to verify the functionality of the Convertor.

import java.util.Scanner;

public class TestConvertor {
    public static void main(String[] args) {
        Convertor myConv = new Convertor();
        Scanner sc = new Scanner(System.in);

        System.out.print("Enter temp in Celsius: ");
        double c = sc.nextDouble();

        // Calling static method (Class name way)
        System.out.println("F (static call): " + Convertor.cToF(c));

        // Calling static method (Instance way - NOT recommended but allowed)
        System.out.println("F (instance call): " + myConv.cToF(c));
        
        // Note: There is no difference in the result.

        System.out.print("Enter temp in Fahrenheit: ");
        double f = sc.nextDouble();
        
        // Non-static method MUST be called via instance
        System.out.println("C: " + myConv.fToC(f));
    }
}

Observation:

While you can call a static method through an object reference (e.g., myConv.cToF()), it is considered bad practice in Java. Always use the class name (Convertor.cToF()) to indicate it is static.
