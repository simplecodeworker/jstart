The Switch Statement

Transition from simple conditional logic to efficient multi-way branching in Java.

Objective: Logic & Syntax
Level: Beginner

Section 1: Introduction

A selection statement that tests a variable against a list of values (cases).

Syntax Guide 
switch (expression) {
    case value1:
        // code if expression == value1
        break;
    case value2:
        // code if expression == value2
        break;
    default:
        // catch-all code
}

Core Rules

  • Supported Types int, short, byte, char, String (since JDK 7), and enums.
  • Exact Matches Only It checks for equality. Boolean expressions like age > 10 are invalid!

Pro Tip: Using Enums

An Enum (Enumeration) is a special data type used to define a collection of constants. Using enums with switch makes your code type-safe and much easier to read.

Do we need to define RED as "red" or 1?

No! In Java, the enum constant itself is the value. You don't need to manually map them to strings or numbers; the compiler handles the unique identification for you.

enum TrafficLight { RED, YELLOW, GREEN }

TrafficLight light = TrafficLight.RED;

switch (light) {
    case RED: 
        System.out.println("STOP"); 
        break;
    case GREEN: 
        System.out.println("GO"); 
        break;
}

Section 3: Practical Examples

Common real-world uses for the switch statement.

1 Numeric Logic: Even or Odd 

int number = sc.nextInt();
switch (number % 2) {
    case 0: 
        System.out.println(number + " is even.");
        break;
    case 1: 
        System.out.println(number + " is odd.");
        break;
}
Teacher's Note: The remainder operator % reduces any number to just two possible outcomes: 0 or 1.

2 String Logic: Weekly Planner 

String day = "Monday";
switch(day) {
    case "Monday":
        System.out.println("Jogging at 5pm");
        break;
    case "Tuesday":
        System.out.println("Ballet at 6pm");
        break;
    default:
        System.out.println("Free activity");
}
Teacher's Note: String comparisons in switch are case-sensitive. "Monday" != "monday".

Section 4: If-Else vs Switch

When to use which tool for the job.

Feature If-Else Switch
Best For Value ranges (score > 90) Discrete values (day == 1)
Data Types Any (specializing in boolean) Restricted (int, String, char)
Complexity Can be messy with many branches Clean, readable, and faster to read

Critical Limitation

If you need to test if a value is within a range (e.g., checking if a temperature is between 30 and 40), you cannot use a switch. Use if-else instead.
