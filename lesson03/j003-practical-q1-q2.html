Java Practical Q3.1 and Q3.2

Q3.1

Lucky Number Guessing

"Write a Java application to allow user to guess your lucky No. Your lucky No is hardcoded in your program. If the user keys in your lucky No, give him a congratulation message, otherwise, show him a 'Thank You' message."

Q3.1(a) Using If-Else Java 
import java.util.Scanner;

public class LuckyGuess {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        int luckyNo = 7; 
        
        System.out.print("Guess my lucky number: ");
        int guess = input.nextInt();
        
        if (guess == luckyNo) {
            System.out.println("Congratulations!");
        } else {
            System.out.println("Thank You");
        }

        // close the scanner to release resources
        input.close();
    }
}

⚠️ Beginner Watch-out

Notice the use of == (double equals). Beginners often use a single =, which is for assignment, not comparison. This will cause a compilation error in Java inside an 'if' condition for integers.

Q3.1(b) Using Switch (with numerical constant) Java 
switch (guess) {
    case 7:
        System.out.println("Congratulations!");
        break;
    default:
        System.out.println("Thank You");
}

⚠️ Beginner Watch-out

In Java, switch case labels must be constant expressions. If you try to use a regular variable in a case, it won't work. By adding final, you tell Java the value will never change, making it valid for the switch.

Q3.1(b) Alternative: Using Switch (with final constant) Java 
// Using 'final' makes the variable a constant, allowing it in case labels
final int LUCKY_NUMBER = 7;

switch (guess) {
    case LUCKY_NUMBER:
        System.out.println("Congratulations!");
        break;
    default:
        System.out.println("Thank You");
}

⚠️ Beginner Watch-out

In Java, switch case labels must be constant expressions. If you try to use a regular variable in a case, it won't work. By adding final, you tell Java the value will never change, making it valid for the switch.

Q3.1(c) Shorthand: (condition)? true : false Java 
String message = (guess == luckyNo) ? "Congratulations!" : "Thank You";
System.out.println(message);

⚠️ Beginner Watch-out

The ternary operator ? : is great for simple assignments, but don't over-nest them. If your logic gets complex, go back to if-else for readability.

Q3.2

Quiz Mark Grading

"Write a program to prompt the user to enter a quiz mark of John. Display a message to show which grade John got (A: 80-100, B: 70-79, C: 60-69, D: 50-59, F: 0-49). (a) use if -- else if – else (b) add some codes to your program so that it can check and display whether the marks entered are valid or not. A valid mark should be between 0 and 100."

Q3.2(a & b) If-Else-If with Validation Java 
if (mark < 0 || mark > 100) {
    System.out.println("Invalid mark! Must be 0-100.");
} else if (mark >= 80) {
    System.out.println("Grade: A");
} else if (mark >= 70) {
    System.out.println("Grade: B");
} else if (mark >= 60) {
    System.out.println("Grade: C");
} else if (mark >= 50) {
    System.out.println("Grade: D");
} else {
    System.out.println("Grade: F");
}

⚠️ Beginner Watch-out

Order matters! When using else if for ranges, start from the highest (80, 70, 60...) or the lowest. If you check mark >= 50 first, a score of 95 would match the first block and display "D".

Q3.2(c1) "If user enters an invalid mark, allow user to enter again until -1 is entered. " with While Loop: Allow re-entry until -1 Java 
System.out.print("Enter mark (-1 to quit): ");
int mark = input.nextInt();

while (mark != -1) {
    if (mark >= 0 && mark <= 100) {
        // ... Grading logic ...
    } else {
        System.out.println("Invalid mark!");
    }
    System.out.print("Enter mark (-1 to quit): ");
    mark = input.nextInt(); // Update mark
}

⚠️ Beginner Watch-out

Infinite Loops: Beginners often forget to re-ask for input inside the loop. Without mark = input.nextInt(); at the bottom of the loop, the value of mark stays the same forever.

Q3.2(c2) "If user enters an invalid mark, allow user to enter again until -1 is entered. " with Do-While Loop: Allow re-entry until -1 Java 
int mark;
do {
    System.out.print("Enter mark (-1 to quit): ");
    mark = input.nextInt();
    
    if (mark == -1) break;
    
    if (mark >= 0 && mark <= 100) {
        // ... Grading logic ...
    } else {
        System.out.println("Invalid mark!");
    }
} while (mark != -1);

⚠️ Beginner Watch-out

The Semicolon: A do-while loop must end with a semicolon after the while(...). Also, notice we check mark == -1 early to avoid printing "Invalid mark" when the user wants to quit.
